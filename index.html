Команды Git

Определение состояния

status — это еще одна важнейшая команда, которая показывает информацию о текущем состоянии репозитория: актуальна ли информация на нём, нет ли чего-то нового, что поменялось, и так далее. Запуск git status на нашем свежесозданном репозитории должен выдать:

        $ git status
        On branch master
        Initial commit
        Untracked files:
        (use "git add ..." to include in what will be committed)
        hello.txt

Подготовка файлов

Если нужно добавить все, что находится в директории, мы можем использовать

        $ git add -A

Если нужно добавить один файл

        $ git add file.txt

Коммит(фиксация изменений)

Ключ -m и сообщение — это созданное пользователем описание 

        $ git commit -m "Initial commit."

Подключение к удаленному репозиторию

Обычно главный репозиторий называется origin.

        $ git remote add origin URL

Отправка изменений на сервер

Имеет два параметра: имя удаленного репозитория (origin) и ветку (master)

        $ git push origin master

Клонирование репозитория

Новый локальный репозиторий

        $ git clone URL

Запрос изменений с сервера

Имеет два параметра: имя удаленного репозитория (origin) и ветку (master)

        $ git pull origin master

Ветвление

Ветви имеют свою собственную историю и изолированные друг от друга изменения до тех пор, пока вы не решаете слить изменения вместе.

Активная ветка помечена звездочкой

        $ git branch
        * master

Создание новой ветки

Чтобы создать еще одну ветку, используем команду branch name

        $ git branch new_branch_name

Переключение между ветками

Для переключения команда checkout

        $ git checkout new_branch_name

Слияние веток

В ветке new_branch_name создан новый файл file.txt

        $ git add file.txt
        $ git commit -m "New feature complete.”

Изменения завершены, теперь мы можем переключиться обратно на ветку master.

        $ git checkout master

Чтобы file.txt появился в ветке master, нужно воспользоваться merge для объединения веток (применения изменений из ветки new_branch_name к основной версии проекта).

        $ git merge new_branch_name

Теперь ветка master актуальна. Ветка new_branch_name больше не нужна, и ее можно удалить.

        $ git branch -d new_branch_name

Отслеживание изменений, сделанных в коммитах

У каждого коммита есть свой уникальный идентификатор

        $ git log
        commit 1bf94010b3e9aa5214fdeceb29b0dd37e45ff61b (HEAD -> master, origin/master)
        Author: Andrey

Чтобы посмотреть, что нового появилось в коммите, мы можем воспользоваться командой show [первых нескольких символов commit]

        $ git show 1bf9
        commit 1bf94010b3e9aa5214fdeceb29b0dd37e45ff61b (HEAD -> master, origin/master)

Чтобы увидеть разницу между двумя коммитами, используется команда diff (с указанием промежутка между коммитами):

        $ git diff 09bd8cc..ba25c0ff

Возвращение файла к предыдущему состоянию

Гит позволяет вернуть выбранный файл к состоянию на момент определенного коммита. Это делается уже знакомой нам командой checkout, которую мы ранее использовали для переключения между ветками. Но она также может быть использована для переключения между коммитами (это довольно распространенная ситуация для Гита — использование одной команды для различных, на первый взгляд, слабо связанных задач).

        $ git checkout 09bd8cc1 hello.txt

Исправление коммита

Самый последний коммит может быть доступен по алиасу HEAD:

        $ git revert HEAD

Для остальных будем использовать идентификаторы:

        $ git revert b10cc123
